# 배열(Array)

---

ArrayStudy1,2,3 를 통해, JAVA의 배열에 대해 알아보았다 <br>
공부한 내용을 정리해 보겠다
---

### 장점
```
1. 구현이 쉽다
2. 인덱스를 이용하여 접근이 가능해 접근속도가 빠르다
```

### 단점
```
1. 크기를 마음대로 수정할 수 없다
    1. 크기를 크게 잡을경우 메모리가 낭비된다
    2. 작게 잡을경우 그 이상 데이터를 저장하지 못한다
2. 메모리의 재사용이 불가능하다
    1. 초기 사이즈만큼 메모리를 할당받아 사용
    2. 데이터의 존재 유무와 관계없이 그 만큼의 메모리를 사용
    3. 배열 자체를 제거하지 않는 이상 메모리의 재사용은 불가능
3. 삽입과 삭제에 비효율적인 구조
    1. 삽입하고자 하는 인덱스 뒤의 원소부터 맨 끝원소까지 한 칸씩 재할당하는 과정이 필요
    2. 삭제하고자 하는 인덱스 뒤의 원소부터 맨 끝원소까지 한 칸씩 재할당하는 과정이 필요
```

### 시간복잡도
```
1. 인덱스가 i인 원소 
    접근 -> O(1)
    수정 -> O(1)

2. 특정원소를 탐색 -> O(n)

3. 추가,삭제 -> O(n) (일반적으로)
    맨앞 원소 삽입 -> O(n)
    맨앞 원소 삭제 -> O(n)
    
    중간 원소 삽입 -> O(n)
    중간 원소 삭제 -> O(n)
    
    맨뒤 원소 삽입 -> O(1)
    맨뒤 원소 삭제 -> O(1)
```

## [1] 배열 (Array)
```
인덱스와 인덱스에 대응하는 데이터들로 이루어진 자료구조

 1. 배열을 이용하면 한번에 많은 메모리 공간을 할당
 2. 같은 타입의 데이터들이 저장
    인덱스를 이용하여 데이터 접근가능
    반복문을 이용하여 처리하게 적합
 3. 배열 생성시, 해당 배열의 주소값을 가지게 된다 (주소값을 통해 데이터에 접근)
 4. 배열을 선언만 하고 생성을 안할 시 주소값은 가지지만, 공간은 할당되지 않는다
 5. 배열을 생성할땐 크기를 지정해주어야 한다
 6. 배열의 인덱스는 0부터 시작하며, 배열의 마지막 인덱스는 (배열크기-1) 이다.
```

```java
//배열의 선언;
int[] arr;

// 배열의 생성
int[] arr2 = new int[5];

// 배열의 생성2 (선언만 된 배열의 크기를 지정해주어 생성)
arr = new int[3];

// 생성과 동시에 초기화 (배열의 크기를 써주지 않아도 자동으로 된다)
int[] arr3 = {1, 2, 3, 4, 5, 6, 7};
```

## [2] 배열의 크기 (length)

```        
배열의 크기를 구하고 싶다면?
직접 세보거나 일일히 확인해서 사용하기엔 실수할 가능성이 높다
그럴땐 간단하게 length를 활용하면된다
```
```java
//size 변수에 arr1.length, 즉 배열의 크기를 넣었다 size를 출력해보면?
String[] arr1 = new String[65465];
int size1 = arr1.length;
String[] arr2 = new String[1];
int size2 = arr2.length;

System.out.println(size1);
//출력결과 : 65465
System.out.println(size2);
//출력결과 : 1
```

## [3] 배열과 반복문
```
배열은 반복문을 이용하여 처리하기에 적합하다

[상황 1] 100개의 크기를 가지는 배열을 만들어, 
각각방에 1부터 100까지 순서대로 숫자를 넣고 출력하려면?
        
각각 데이터를 넣기엔 시간도 오래걸리고
실수할 가능성이 굉장히 높다 (인덱스나 값을 잘못 넣는다던지...)

그럴땐 반복문을 활용하면 알아서 잘 넣어주고 출력해준다 
```
```java
int[] arr4 = new int[100];
for (int i = 0; i < arr4.length; i++) {
    arr4[i] = i + 1;
}

for (int i = 0; i < arr4.length; i++) {
    System.out.println(arr4[i]);
}
```

## [4] 2차원배열
```
2차원배열이란 배열의 요소가 1차원 배열을 가지는 배열을 말한다.
선언 및 생성은 1차원 배열과 비슷하다
배열을 선언하면 기본적으로 모든 요소 값은 0으로 설정된다.

일반적으로
int[][] arr = new int[rows][columns]; 의 형태로 생성해준다
아래 예시를 보며 빠르게 이해해보자
```
```java
int[][] arr = new int[2][3];
```
```
위의 배열은 이해하기 쉽게 그려보면

------ Column0  columns1  columns2
Row 0   [0][0]   [0][1]   [0][2]
Row 1   [1][0]   [1][1]   [1][2]

이런 형태가 된다


<추가설명>

-- 전에 정리한 일반적인 배열(1차원배열)을 다시 봐보자
int[] arr = new int[3]
arr[0] = 어떤값;
arr[1] = 어떤값;
arr[2] = 어떤값;
이렇게 되는건 무리없이 이해할수 있을것이다

--- 2차원배열은
int[][] arr = new int[2][3]
arr[0] = 1차원배열; -> 크기는 3
arr[1] = 1차원배열; -> 크기는 3
이런 형태라 보면된다

그렇다고 마구잡이로 합치면 안된다
각각 배열의 목적이 있을것이고, 2차원배열을 봤을때 직관적인 이해가 어려울수 있으므로
적절한 상황에서 사용하자
```

## [5] 2차원배열의 크기
```java
// 이 2개의 배열은
int[] arrExample1 = new int[3];
int[] arrExample2 = new int[3];

//이 2차원배열과 같은 형태다
int[][] arrExample3 = new int[2][3];
```
```
위의 2차원 배열을 예시로 크기를 구해보려한다
arrExample3.length 는 2가 되며
arrExample3[i].length 는 3이 된다
```
```java
System.out.println(arrExample3.length);
//출력결과 : 2
System.out.println(arrExample3[0].length);
//출력결과 : 3
```
